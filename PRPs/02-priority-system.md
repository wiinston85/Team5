# PRP 02: Priority System

## Feature Overview
Add high, medium, and low priority across API and UI with sorting and filtering.

## Technical Requirements
- Priority type: high | medium | low.
- Database default priority: medium.
- API input validation for priority.

## UI Requirements
- Priority dropdown in create and edit flows.
- Color badges:
  - High: red
  - Medium: amber
  - Low: blue
- Priority filter dropdown.

## Acceptance Criteria
- Three priorities work end-to-end.
- Todo sorting order: high, medium, low.
- Filtering by priority works.

## Testing Requirements
- E2E for create/edit/filter/sort.
